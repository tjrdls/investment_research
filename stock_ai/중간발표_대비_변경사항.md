# 중간발표(stockAI) → 현재(stock_ai) 변경 정리

> **`stock_ai` 는 원본 프로젝트의 개선본입니다.** 중간발표에서 보여준 멀티모달 분석 시스템을
> 버리지 않고, 그 정체성을 **유지**하면서 (1) 검증 가능성, (2) 데이터 정합성,
> (3) 리스크 관리, (4) 운용 인프라를 더한 연속적 발전입니다.
>
> **코드 계보**: 원본/중간발표 = `investment_research` (**멀티모달 LSTM, early fusion**) →
> `stockAI` (late fusion 전환·텍스트 분리한 plain LSTM) → `stock_ai` (lateLSTM → LGBM, 현재).
> 이 문서는 **중간발표(원본) vs 현재(stock_ai)** 대비입니다.

---

## 0. 한눈 요약 — 정체성의 "확장"

| | 중간발표 (stockAI) | 현재 (stock_ai) |
|---|---|---|
| 핵심 정체성 | **멀티모달 단일 종목 분석** (차트+재무+뉴스 → 매수/관망/매도) | **멀티모달 분석은 유지** + 검증 가능한 **포트폴리오 시스템** 추가 |
| 딥러닝 모델 | **멀티모달 LSTM** (LSTM 차트 + DART 텍스트 임베딩 concat, **early fusion**, 3분류) | **LGBM Hybrid**(현재) + LSTM 계열 병행 (그 사이 **lateLSTM** 양방향+attention 으로 개선) |
| 출력 | 한 종목 의견 (텍스트) | 한 종목 멀티모달 분석 **＋** Top10 포트폴리오 |
| 검증 | 단순 분류 정확도 | walk-forward · 일별 MDD · Monte Carlo · 블랙스완 |
| GUI | 단일 종목 분석 화면 | 8개 탭 (**🧬 멀티모달** 탭이 첫 화면) |

> 한 줄: **"AI를 어떻게 쓸까"(중간) → "무엇을 믿고 무엇을 의심할까"(현재)**.
> 멀티모달은 그대로, 그 위에 *정직한 검증*이 얹혔습니다.

## 1. 모델 — 두 단계 서사: ① LSTM 개선 → ② LGBM 으로 변경

> **호칭**: 발표/문서에서는 차트 전용 LSTM 을 **`lateLSTM`** 으로 부릅니다. (코드 클래스명은
> `ChartLSTM` 이지만 "Chart"가 *주가 차트* 의미라 트리/그래프 LSTM 과 혼동될 수 있어, 텍스트를
> 쓰던 원본 멀티모달 LSTM 과 구분하는 발표용 호칭으로 `lateLSTM` 사용.)

### 서사 ① — LSTM 개선 (멀티모달 LSTM → lateLSTM → 개선판)
- **원본 / 중간발표 (investment_research)** — **멀티모달 LSTM** (`MultimodalStockPredictor`):
  LSTM(차트 시퀀스 11피처) + DART 재무 **텍스트 임베딩(1536-d)** 을 **모델 내부 concat (early fusion)** → 3분류.
- **late fusion 전환** — 텍스트를 모델 밖으로 분리 → 차트 전용 **lateLSTM**(단방향, 3분류)으로.
- **lateLSTM 개선** — **양방향(Bidirectional) + Temporal Attention + 회귀 듀얼헤드**로 업그레이드
  (~162K, 8 차트피처, 126일 회귀). 검증 IC(val) **0.227**.

### 서사 ② — LGBM 으로 변경 (누수 발견 → 트리 채택)
- 진짜 **walk-forward**(시간순 + 42일 갭) 재검증 시 lateLSTM **OOS IC 0.04 로 폭락** →
  검증셋 미래정보 누설(**look-ahead 누수**) 발견.
- **LightGBM(v3 차트 10피처 / v4 차트+펀더 13피처)** 교체 → OOS Rank IC **0.13** (≈3배 알파, 1/30 시간).
- **LSTM 계열은 제거하지 않고 병행** — "딥러닝이 언제 유리/불리한가" 비교 기준:

| 실행 | 모델 |
|---|---|
| `backtest --strategy chart-lstm` | lateLSTM (개선판; 코드 클래스 `ChartLSTM`) |
| `backtest --strategy ensemble --ai-weight 0.2` | LGBM Hybrid (v3 60% + v4 40%) |
| `backtest --strategy rule` | 규칙 4팩터 (AI 미사용) |

> **발표 포인트**: "lateLSTM IC 0.227(검증) → 0.04(정직한 OOS) 붕괴 → 누수 추적 → LGBM 채택"
> 이라는 *실패에서 배운 서사*. LSTM 계열은 대조군으로 남습니다.

## 1-B. 예측 타깃·호라이즌의 변천 (원리: **라벨 = 실제 보유기간**)

| 시점 | 타깃 호라이즌 | 형태 | 왜 |
|---|---|---|---|
| **원본 멀티모달 LSTM** (investment_research) | **5일 뒤** | 상승/하락/횡보 **3분류** (±2%) | 단일 종목 *단기 방향* 예측 |
| lateLSTM (개선판) | **126일 (6개월)** | 회귀 + confidence | 그 시절 **6개월 리밸런싱** 보유기간에 정합 |
| **stock_ai 현재 (LGBM v3/v4)** | **42일 (2개월)** | **횡단면 백분위 랭킹** | 리밸 **2개월 최적화** + Top-N *상대* 선별 |

**왜 5일이 아니게 됐나**: 5일 예측은 단일 종목 단기 방향용. 시스템이 *몇 달 보유하는 포트폴리오*로
바뀌자, 5일 뒤를 맞춰도 행동에 안 쓰임 → 라벨을 **보유기간**에 맞춰야 함.

**왜 lateLSTM 은 126일인가**: 6개월 리밸 시절 모델이라 보유기간(126일)에 맞춘 것. 이후 리밸이
2개월로 최적화되고 메인 모델이 LGBM 으로 교체되며 42일이 됐으나, **lateLSTM 은 그 전에 폐기
라인으로 빠져 126일에 멈춤** (의도적 "LSTM=126 최적" 결론이 아니라 *재학습 안 된 잔존값*).

**왜 분류 → 횡단면 랭킹인가**: Top-N 을 *상대 강도*로 고르므로 "오를까/내릴까"(절대 방향, 시장
베타에 휘둘림)가 아니라 "같은 날 동종 대비 더 오를까"(백분위)가 학습 가능한 신호. 평가=Rank IC.

> **🔬 열린 항목 (검증 예정 — ① 비교 하네스에서)**
> lateLSTM 을 42일로 재학습 시 성능 저하 여부는 **미측정**. 같은 walk-forward 에서
> lateLSTM @42 vs @126 을 Rank IC·decile spread 로 측정해야 함.
> a priori 트레이드오프: 42일 = 샘플↑(꼬리 42일만 손실, LSTM 에 유리)·보유기간 정합 vs
> 라벨 노이즈↑; 126일 = 평활 효과↑·샘플↓·현 2개월 리밸과 불일치. **실측 전 단정 불가.**
> ⚠️ 모델 비교는 반드시 **동일 42일 타깃으로 통일**해야 공정 (호라이즌 차이를 모델 차이로 오인 방지).

## 2. 융합 — Early(Intermediate) Fusion → Late Fusion (+ 학습 게이팅 옵션)

### 원본 / 중간발표 (investment_research)
- **Early/Intermediate Fusion**: `MultimodalStockPredictor` 가 LSTM(차트) + 텍스트 임베딩을
  **모델 내부에서 concat** 해 단일 분류기로 출력 → "왜 그 결론인지" 설명 불가(블랙박스).

### 전환 → Late Fusion (의도적 결정)
- 모델 내부 결합을 버리고, 각 모달이 **독립 신호**를 내고 모델 밖에서 결합 → 신호 충돌 감지·근거
  설명 가능. (early 가 나빠서가 아니라 *late 가 더 나아서* 바꾼 것.)

### 현재 (stock_ai)
- **Late Fusion 철학 유지** — 각 모달이 독립 신호(0~100)를 내고 해석 가능하게 결합.
  `src/modality/` 로 표준화: `ModalitySignal` 단위, 5개 모달(차트·펀더·밸류·뉴스·매크로).
- **기본은 가중 평균 fusion** (해석 가능·과적합 없음·결측 우아 처리) — *이건 결함이 아니라 의도된 설계*.
- **추가 옵션**: 학습된 **게이팅 신경망**(`LearnedFusion`) — 모달 가중치를 고정하지 않고
  신호 패턴(점수·신뢰도·가용성)에 따라 **조건부로 학습**. 모델 없으면 가중 평균으로 자동 폴백.
  - 학습: `python main.py train-fusion` (타깃 = 42일 후 횡단면 백분위)
  - 뉴스는 과거 재구성 불가 → 학습 제외, 추론 시 고정 비중 blend.

> 가중 평균(해석) ↔ 학습 게이팅(적응) **둘 다 제공**. GUI 라디오로 즉시 비교 가능.

## 2-A. 단일 종목 분석 → 종목 추천(포트폴리오)으로 확장 — 왜?

**원본 (investment_research)**: **단일 종목 분석** — 종목 코드 1개 입력 → 매수/관망/매도 텍스트 의견.

**한계 (확장 이유)**:
- "이 종목 어때?"엔 답해도 **"내 돈을 *어디에 얼마나*?"엔 못 답함** (한 종목 의견만).
- 출력이 **텍스트 의견**이라 **검증 불가** — 맞았는지 측정할 백테스트가 성립 안 됨.

**확장 (stock_ai)**: 전 종목에서 **Top N 을 골라 비중까지** 결정 = **포트폴리오**.
- → **백테스트 가능** → CAGR·Sharpe·일별 MDD·블랙스완·몬테카를로로 **검증 가능**.
- → 단일→추천 확장의 핵심 이유 = **"검증 가능성"**. 이 프로젝트의 *정직한 검증* 정체성이 여기서 시작.

**단일 종목 분석은 버리지 않음** — 🧬 멀티모달 탭으로 유지. 즉 **두 출력 모두 제공**:
한 종목 깊은 해석(단일) + 전 종목 줄세워 상위 선별(추천).

> 추천 파이프라인: 하드필터 → Rule 점수(+모멘텀) → **LGBM AI 점수** → 앙상블(0.2·AI + 0.8·Rule)
> → 코스피5+코스닥5. 즉 추천의 `ai_score` = **LGBM v3/v4** (lateLSTM 아님).

## 2-B. 추천도 멀티모달로 — 점수 + 뉴스 텍스트 → GPT 종합

**문제 인식**: 종목 추천/백테스트의 `ensemble = 0.2·AI + 0.8·Rule` 은 **둘 다 숫자 모델** →
late fusion 이긴 하나 **멀티모달이 아님**(텍스트·GPT 없음). 멀티모달은 단일 종목 분석에만 있었음.

**해결 — GPT 가 (점수 + 텍스트)를 종합** (`src/recommend/multimodal_recommend.py`):
- **[1단]** Rule+AI **숫자** 스크리닝(`select_top_n`)으로 Top N 선별 — 싸고 빠르고 **백테스트 가능**.
- **[2단]** picks 의 **이미 계산된 값**(앙상블/Rule/AI 점수·ROE·PER 등) + 종목별 **뉴스 텍스트 원문**을
  **그대로 GPT 에 전달** → GPT 가 종합 정리. **재평가·2차 정량 fusion 없음** (GPT 가 fusion 역할).

**왜 이 구조인가** (채택 근거):
- 멀티모달성 = **숫자 모달(점수) + 텍스트 모달(뉴스)을 GPT 가 결합** — 원본 stockAI 의 "LLM 이 신호+텍스트
  읽고 종합 판단" 철학과 동일. (정량 점수는 그대로 두고 GPT 는 *해석·정리*만.)
- 뉴스·GPT 는 **과거 재구성 불가(look-ahead)** → 백테스트엔 못 넣음 → **백테스트=숫자(전략 검증) /
  라이브 추천=점수+뉴스+GPT(오늘 뭐 살까)** 역할 분리.
- GPT 종합은 `gpt-5.4-nano`(없으면 규칙 템플릿 fallback). GUI 추천 탭 하단 **"🧬 멀티모달 종합(점수+뉴스→GPT)"** 버튼.

## 3. 데이터 — 단일 종목 → 전종목 + 자체 TTM PER + 버그 3건 수정

### 중간발표 (stockAI)
- 상위 소수 종목 중심, KRX 공식 지표 사용.

### 현재 (stock_ai)
- **전 종목(KOSPI+KOSDAQ) OHLCV + 시총 + DART 분기 재무**.
- **자체 TTM PER**: KRX 공식 PER은 EPS 연 1회 갱신이라 이익 급증주가 부풀려 보임 →
  DART 분기 순이익 4개 합산으로 직접 계산 (공시일 분기말+45/90일로 룩어헤드 차단).
  - 효과: 총수익 408.6% → **466.0%**, 샤프 0.86 → **0.90**, MDD -25.2% → **-22.1%**.
- **발견·수정한 데이터 버그 3건** (발표 하이라이트):
  1. `compute_fundamentals` 누계/단독 오류 (Q1~Q3 도 누계로 처리) → Q4 만 누계로 수정.
  2. `ACCOUNT_MAP` '영업수익' 누락 (SK하이닉스/금융주 매출 NULL) → **2,212행 매출 회수, TR 587% → 1,427%**.
  3. YoY 분모 음수 처리 (적자→흑자 턴어라운드 미캡처) → `(curr-prev)/abs(prev)` 로 수정.

> 모델보다 **데이터 정합성**이 알파에 더 컸다는 것 (②번 하나로 알파 약 3배)이 핵심 교훈.

## 3-B. 수익성 — 가격 모멘텀 팩터 추가 (4팩터 → 멀티팩터)

Rule 점수에 **12-1개월 가격 모멘텀**(최근 1개월 제외 = 단기 반전 회피)을 5번째 백분위 팩터로
추가(기본 가중 0.20, 자동 재정규화). `ScreenerWeights.price_momentum`, `select_top_n(use_momentum=True)`.

- **채택 근거 — 평균(기대수익)을 높이는 선택**: 횡단면 모멘텀은 특정 강세장 베팅이 아니라
  여러 레짐에서 작동하는 **지속적 위험 프리미엄**이라, 고점(변동성)이 아니라 기대수익의
  *평균*을 끌어올린다.
- **멀티팩터 효과**: 단일 팩터 부진 구간을 다른 팩터가 분산 → 수익 곡선이 매끄러워짐(일관성↑).
- **즉시 구현 가능**: OHLCV 만으로 산출(추가 데이터·승인 불필요).
- LGBM 앙상블 후보 풀에도 자동 반영(`LGBMEnsembleScreener.select_top_n(use_momentum=True)`).
- 재현: 재현 탭 ⑥-B (모멘텀 OFF vs ON).

## 5. 리스크 관리 — 없음 → 구조적 방어

### 중간발표
- 종목 의견만 제시, 포지션·방어 로직 없음.

### 현재 (`backtest`/`risk_overlay`)
- **3-state 분할 스위칭**: Ichimoku 구름대 + ADX/DMI 신호로 주식 100% → 50% → 0%.

## 6. 운용 인프라 (현재 단계에서 추가)

중간발표 이후 "실제로 돌아가는 시스템"으로 만들기 위한 작업:

- **KRX OpenAPI 로더** (`KRX_AUTH_KEY`): KRX 사이트 로그인 게이트 우회. pykrx가 막힌
  유니버스·시총·증분 업데이트를 공식 REST API(날짜-bulk)로 대체.
- **시드 모드** (`main.py seed`): KRX 승인 전, 하드코딩 우량주 26종목으로 전체 파이프라인을
  즉시 구동 (OHLCV=pykrx, 상장주식수=yfinance). 중간발표의 "소수 종목" 정신과 연속.

- **CLI 정비**: `--strategy ensemble` 을 (레거시 lateLSTM → ) **LGBM Hybrid** 로 재배선,

### 🔁 재현(Reproducibility) 탭 신설 — **변경별 before→after 재현**
각 항목 제목 = **"무엇에서 무엇으로 바꿨는가"**, 본문 = 기술 설명 + 그 변경을 현재 코드로
직접 재현(before/after 표). "정직한 검증"을 GUI로 증명.

| 항목 (제목 = 전환) | 재현 |
|---|---|
| ① 메인 모델: 멀티모달 LSTM → lateLSTM → LGBM | 🔬 비교 하네스 후 |
| ② 예측 타깃: 5일·3분류 → 42일·횡단면 랭킹 | 🔬 비교 하네스 후 |
| ③ 융합: Early → Late (+학습 게이팅) | ✅ 가중평균 vs 학습게이팅 (early 비교는 🔬) |
| ④ 밸류에이션: KRX 공식 PER → 자체 TTM PER | ✅ use_ttm_per OFF vs ON 백테스트 |
| ⑤ 리밸런싱: 6개월 → 2개월·매월 9일 | ✅ 주기 그리드 (1/2/3/6m) |
| ⑥ AI 비중: 50%·30% → 20% | ✅ ai_weight 그리드 (0~1.0) |
| ⑥-B 팩터: 4팩터 → +12-1개월 가격 모멘텀(멀티팩터) | ✅ 모멘텀 OFF vs ON 백테스트 |
| ⑦ 리스크: 무방어/손절 → Ichimoku·ADX 분할+금헤지 | ✅ 방어 OFF vs ON 백테스트 |
| ⑧ 검증: 단순 → walk-forward·일별MDD·블랙스완·MC10k | ✅ 블랙스완+MC 재현 |

- 백테스트형 항목은 공유 기간(시작/종료연도)으로 실행. 현재 DB가 **시드/전체인지 자동 감지** →
  시드면 "KRX 승인 후 같은 버튼이 진짜 수치 재현" 안내.
- 각 본문의 "〔여기에 기술 설명〕" 자리는 발표 준비 시 상세 기술 내용을 채우는 용도.

> **발표 메시지**: "우리가 바꾼 모든 결정은 재현 탭에서 *before→after* 로 클릭 재현됩니다."

### 성능 — 무거운 탭 지연 로딩 (콜드 스타트 개선)
`st.tabs` 가 매 실행마다 모든 탭 본문을 돌려 성과·검증의 풀 백테스트·MC10k 가 초기에 다 돌던 문제를
**버튼 게이트(`_lazy_gate`)** 로 해결 — 해당 탭에서 버튼을 눌러야 계산(이후 캐시되어 즉시).
캐시 자체는 정상 동작 확인(중복 백테스트 없음: `_compute_validation` 이 `get_live_backtest` 재사용,
캐시 키 = DB 최대일로 안정).

## 9. 발표용 핵심 메시지

1. **버린 게 아니라 확장했다** — 멀티모달·lateLSTM 둘 다 유지, 그 위에 검증·포트폴리오·방어를 추가.
2. **정직한 검증이 본질** — LSTM IC 0.227 → 0.04 누수 발견이 전환점. "딥러닝은 도구일 뿐."
3. **데이터 정합성이 모델보다 컸다** — 버그 ②번 하나로 알파 약 3배.
4. **둘 다 제공** — 모델(LSTM/LGBM), 융합(가중/학습), 데이터(전종목/시드) 모두 선택 가능한 시스템.
