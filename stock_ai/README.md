## 1. 한눈에 보는 전체 흐름

이 시스템은 "좋은 회사를, 좋은 시점에, 적절히 분산해서, 위기에는 방어하며" 담는 것을 자동화합니다.

```
[데이터 수집]  KRX 주가·시총 + DART 재무제표 + PER/PBR
        │
        ▼
[1단계: 하드 필터]  통과 못 하면 후보에서 탈락
   시총 1조+ · ROE 20%+ · PER 60↓ · 영업이익률 10%+ · 매출/순익/영업이익 성장 > 0 · 일회성이익 차단
        │
        ▼
[2단계: 추세 필터]  종가 > 60일선 (매수 시점에만 적용)
        │
        ▼
[3단계: Rule 점수]  PER·ROE·매출성장·순익성장 4팩터 백분위 + 시총/성장 가산
        │           → 상위 N×3 후보 풀
        ▼
[3-B: AI 점수]  LightGBM Hybrid (v3 차트 + v4 차트+펀더멘털)
        │           ensemble = 0.2×AI + 0.8×Rule
        ▼
[4단계: 포트폴리오]  상위 10종목 (코스피 5 + 코스닥 5)
   rank 가중 (1위 ~18-20%, 꼴찌 ~2%) · 단일종목 25% 캡 · 초과분은 채권(연 3%)
        │
        ▼
[5단계: 리밸런싱·거시 방어]  2개월마다(매월 9일) 재선정
   Ichimoku 구름대 + ADX 신호로 주식 100%→50%→0% 분할 스위칭, 약세장엔 금 ETF
        │
        ▼
[결과]  대시보드에서 성과·보유 포트폴리오·검증을 실시간 확인
```

## 3. 데이터 — 무엇을 어디서 모으나

| 데이터 | 출처 | 테이블 | 용도 |
|---|---|---|---|
| 일별 주가 (OHLCV) | KRX (pykrx) | `ohlcv` | 차트 피처·수익률 계산 |
| 시가총액 | KRX (pykrx) | `market_cap` | 시총 필터·가산점 |
| 분기 재무제표 | DART OpenAPI | `financials` | 매출·영업이익·순이익·자본 |
| 재무 지표 | DART 가공 | `fundamentals` | ROE·영업이익률·성장률 |
| PER/PBR | KRX + 자체 TTM | `per_history` | 밸류에이션 필터·점수 |
| 미국 주가 | yfinance | `ohlcv` | AI 모델 학습 데이터 보강 |

**핵심 포인트 — 자체 TTM PER**
KRX 공식 PER은 EPS를 연 1회(3~5월)만 갱신해, 이익이 급증한 종목의 PER이 과거 기준으로 부풀려 보입니다.
이 시스템은 DART에서 받은 **분기별 순이익 4개를 합산(TTM)**해 PER을 직접 계산합니다.
공시일은 분기말+45일(Q1~Q3), 연말+90일(Q4)로 추정해 **미래 데이터를 미리 쓰지 않도록(look-ahead 차단)** 합니다.

> 예) 에이피알: KRX 공식 PER 88배(미갱신) → 자체 TTM PER 40.7배(실제 이익 반영). 60 기준에서 탈락 대신 통과.

**증분 업데이트**: `data/daily_update.py`는 DB 마지막 날짜 다음날부터 어제까지를 *일자별 bulk API*로 받아 분 단위에 채웁니다. 대시보드의 "데이터 업데이트" 버튼이 이걸 호출합니다.

## 4. 하드 필터 — 원리와 구조

**원리**: "본업이 멀쩡하게 성장하는 우량 기업"만 후보로 남깁니다. 하나라도 못 넘으면 즉시 탈락(AND 조건)입니다.
구현은 `src/screener/rule_based.py`의 `_apply_hard_filter()`입니다.

| 조건 | 기준 (production) | 의미 |
|---|---|---|
| 시가총액 | ≥ 1조 원 | 유동성·안정성 확보 (중·대형주) |
| ROE | ≥ 20% | 자본 대비 높은 수익성 |
| PER | 0 < PER ≤ 60 | 적자 제외 + 과도한 고평가 제외 (자체 TTM PER) |
| 영업이익률 | ≥ 10% | **본업** 수익성 (영업외 이익에 기대지 않음) |
| 매출 성장 (YoY) | > 0 | 외형 성장 |
| 순이익 성장 (YoY) | > 0 | 이익 성장 |
| 영업이익 성장 (YoY) | > 0 | 본업 성장 추세 |
| 일회성 이익 차단 | 0.3 ≤ (순익/영업익) ≤ 2.0 | 자회사 매각 등 일회성 이익으로 부풀린 종목 제외 |

- **YoY 성장률**은 적자→흑자 전환도 잡도록 분모에 절댓값을 씁니다.
- 성장률·ROE·영업이익률은 모두 **TTM(최근 4분기 합)** 기준으로 그 자리에서 재계산해, DART 누계(Q4) 처리 오류와 공시 지연 문제를 동시에 피합니다.
- 기본값은 `config.py`의 `HardFilter`(ROE 20·PER 60·시총 5천억)이며, production 백테스트는 **시총 1조**로 상향(override)해 운용합니다.

## 5. Rule 점수 — 4팩터 + 가산·감점

하드 필터를 통과한 종목끼리 **상대 비교**해 0~100점을 매깁니다 (`_compute_scores()`).
각 지표를 같은 시점 다른 종목과 비교한 **백분위(0~1)**로 바꾼 뒤 가중 평균합니다.

**기본 4팩터 가중치**

| 팩터 | 가중치 | 방향 |
|---|---|---|
| PER | 30% | 낮을수록 ↑ |
| ROE | 30% | 높을수록 ↑ |
| 매출 성장 YoY | 20% | 높을수록 ↑ |
| 순이익 성장 YoY | 20% | 높을수록 ↑ |

(PER 데이터가 없으면 PER을 빼고 나머지 가중치를 자동 재정규화)

**가산·감점 (곱셈 배수)**

| 항목 | 효과 |
|---|---|
| 시총 가산점 | 10조+ ×1.20 · 5~10조 ×1.15 · 2~5조 ×1.08 |
| 영업이익 가속 (YoY) | +100%↑ ×1.30 · +50%↑ ×1.20 · +20%↑ ×1.10 · 감소 ×0.90 |
| 실적 둔화 페널티 (QoQ) | 매출·순익 둘 다 둔화 ×0.85 · 하나만 ×0.92 |
| 추세 강도 가산(옵션) | 5/20/60 정배열 정도에 비례해 가산 |

→ 최종 `rule_score`로 정렬해 **상위 N×3개 후보 풀**을 만듭니다(앙상블 입력).

## 6. AI 모델 — 설명·원리·학습 과정

### 무엇을 예측하나

모델은 주가를 직접 맞히지 않습니다. 대신 매 거래일, **"이 종목이 같은 날 다른 종목들 대비 향후 42거래일(약 2개월) 동안 얼마나 더 오를지"의 순위(랭킹)**를 예측합니다.
이를 위해 학습 라벨로 **`target_rank`** = (같은 날짜의 모든 종목을 대상으로 한 42일 후 수익률의 백분위)를 씁니다. 회귀(LightGBM Regressor)로 이 백분위를 맞춥니다.

### 왜 LightGBM인가

초기에는 LSTM(딥러닝)을 썼지만, 진짜 walk-forward로 재검증하니 시간 누수(leakage)가 있었고 IC가 0.04까지 떨어졌습니다. **LightGBM(그래디언트 부스팅 트리)** 으로 바꾼 뒤 약 3배 높은 알파를 1/30 학습 시간에 얻어 production으로 채택했습니다. legacy LSTM 코드는 기록용으로만 남아 있습니다.

### 두 모델 — v3 / v4

**v3 — 차트 10피처** (`trend_lgbm_v2.py` 기반)

| 분류 | 피처 |
|---|---|
| 추세/위치 (8개) | price_to_ma200, ma60_slope, ma20_to_ma60, bb_position, volume_ratio, macd_hist_ratio, stoch_60_pos, obv_slope_20 |
| 시장 매크로 (2개) | index_ma60_slope, index_mdd_20 (KR=KOSPI200/069500, US=QQQ) |

**v4 — 차트+펀더멘털 13피처** = v3의 10개 + 펀더 3개

| 신규 펀더 피처 | 의미 |
|---|---|
| roe_latest | 공시된 최근 분기 ROE (기업 체급) |
| op_margin_growth | 분기 영업이익률 변화 (실적 모멘텀) |
| per_inverse | 1/PER 안정화 (이익수익률, 밸류에이션) |

- 펀더 피처는 **publish_date(분기말+45/90일) 이전 값만** 매핑(`merge_asof`)해 look-ahead를 차단합니다.
- 미국 종목은 펀더가 없어 NaN이며, LightGBM이 NaN을 자동 처리하므로 v4에서도 학습 universe로 활용됩니다.

### 학습 과정 (walk-forward 시간 분할)

미래 정보가 과거 학습에 새지 않도록 **시간순으로 자르고, 사이에 42거래일 갭**을 둡니다(타깃 horizon만큼).

```
TRAIN  2015-01-01 ~ 2021-12-31     (KR + US 통합)
  ── gap ~42거래일 ──
VAL    2022-03-01 ~ 2023-12-31     (조기 종료 기준)
  ── gap ~42거래일 ──
TEST   2024-03-01 ~ 2026-05-31     (완전 out-of-sample)
```

- 알고리즘: `LGBMRegressor` (learning_rate 0.03, max_depth 6, num_leaves 31, min_data_in_leaf 200, n_estimators 500, early stopping 30)
- 평가 지표: **Rank IC**(예측-실제 순위상관, Spearman), **Spread**(예측 상위 10% 실수익 − 하위 10% 실수익)

**TEST(완전 out-of-sample) 성능**

| 모델 | KR Rank IC | KR Spread |
|---|---|---|
| v3 | **0.1310** | +5.39%pt |
| v4 | 0.1171 | **+6.39%pt** |
| Hybrid(0.6·v3+0.4·v4) | 0.1290 | +5.93%pt |

> IC 0.13은 주식 횡단면 예측에서 상당히 강한 신호입니다(통상 0.05+면 유의미). v3는 전체 순위 정확도가, v4는 극단(최상/최하) 분리가 강해 둘을 섞으면 시너지가 납니다.

학습 실행: `python -m src.model.trend_lgbm_v4` (v3는 v2 베이스에서 생성)

## 7. 앙상블 — 규칙과 AI를 합치는 법

`src/recommend/lgbm_ensemble.py`의 `LGBMEnsembleScreener`가 규칙 점수와 AI 점수를 결합합니다.

```
1. Rule 4팩터로 상위 N×3 후보 풀 생성
2. 각 후보의 13피처 계산
3. KR:  hybrid_ai = 0.6 × v3.predict(10피처) + 0.4 × v4.predict(13피처)
   US:  hybrid_ai = v3.predict(10피처)            (펀더 없음 → v3 단독)
4. ensemble_score = ai_weight × hybrid_ai + (1 − ai_weight) × rule_score
5. ai_weight = 0.2  (production)
```

**왜 AI 비중을 20%만 쓰나** — ai_weight 민감도 그리드(Hybrid 기준):

| ai_weight | 총수익 | 샤프 | 알파/년 |
|---|---|---|---|
| 0.0 (Rule만) | 1427% | 1.41 | +10.31% |
| **0.2 ★** | **1451.9%** | 1.42 | **+10.49%** |
| 0.4 | 1449.4% | 1.42 | +10.47% |
| 0.6 | 1348.7% | 1.40 | +9.72% |
| 0.8 | 1432.0% | **1.44** | +10.35% |
| 1.0 (AI만) | 1164.7% | 1.38 | +8.21% |

검증된 Rule 4팩터가 이미 강한 알파를 내고, AI는 "미세 변별력"만 더하는 역할이라 **20%가 두 신호의 sweet spot**입니다. AI 비중이 0.6 이상이면 Rule의 강한 신호를 희석시켜 알파가 떨어집니다.

## 8. 종목 선별 전체 파이프라인

위 단계를 한 번의 종목 선정(`select_top_n`)으로 묶으면:

```
유니버스(전 종목)
  → [시총·ROE·PER·영업이익률·성장·일회성] 하드 필터        ── 수백 개에서 수십 개로
  → [종가 > 60일선] 추세 필터                              ── 매수 시점 추세만 통과
  → Rule 4팩터 점수 → 상위 N×3 후보 풀
  → 후보별 AI Hybrid 점수 계산 → ensemble_score
  → 코스피 상위 5 + 코스닥 상위 5 = 최종 10종목
```

추세 필터는 **매수 시점에만** 적용합니다. 매수 후 60일선이 깨져도 다음 리밸런싱까지 보유합니다(개별 종목 손절은 백테스트에서 알파를 크게 깎아 도입하지 않음 — 11번 항목 참조).

## 9. 포트폴리오 구성·리스크 오버레이

선정된 10종목을 어떻게 담는가 (`backtest/risk_overlay.py`):

- **rank 가중**: 점수 1위에 가장 큰 비중(약 18~20%), 마지막 종목에 가장 작은 비중(약 2%).
- **단일 종목 25% 캡**: 한 종목이 25%를 넘지 못하게 잘라, 한 종목 사고가 포트폴리오를 무너뜨리지 못하게 함.
- **초과분 → 채권(연 3% 일복리)**: 캡으로 남은 비중과 종목이 0개인 구간은 채권 수익률로 대체.

이 오버레이는 백테스트 결과(`RebalanceResult`)에 사후 적용되어 일별 자산곡선을 다시 계산합니다.

## 10. 리밸런싱·거시 방어 (Ichimoku·ADX·금 헤지)

- **리밸런싱 주기**: 2개월마다 **매월 9일**(주말이면 다음 거래일)에 종목을 재선정.
  9일은 DART 분기보고서 공시 시즌(분기말+45일) 직전이라 신선한 실적을 반영하기 좋은 시점입니다.
- **종목 유지 규칙(keep_simple)**: 기존 보유 종목이 여전히 필터를 통과하면 불필요한 교체를 줄입니다.
- **거시 방어 — 3-state 분할 스위칭** (KOSPI200 069500 일봉 기준):

| 신호 | 포지션 |
|---|---|
| 종가 > 구름대 상단 AND ADX > 25 AND +DI > −DI | 주식 100% |
| 종가가 구름대 안으로 하향 (추세 균열) | 주식 50% / 금 50% (부분 익절) |
| 종가 < 구름대 하단 (완전 이탈) | 주식 0% / 금 100% |

- 약세 구간의 방어 자산은 **현금이 아니라 KODEX 골드선물(H) 132030**입니다. 위기에 음의 상관을 갖는 금이 "데드머니 현금"을 대체해 추가 알파를 만듭니다(Crisis Alpha 근거).
- ADX ≤ 25(약한 추세)일 때는 가짜 신호로 보고 포지션을 유지해 휩쏘를 막습니다.
- **거래비용 0.25%**(단방향)를 모든 매매에 반영합니다.

## 11. 백테스트 결과 — 수익률·MDD·샤프

`src/backtest/rebalance.py`가 매 리밸런싱 구간의 보유·수익률을 시뮬레이션하고, 일별 자산곡선으로 지표를 계산합니다.

**production 시스템 (Hybrid LGBM + Rule, ai_w=0.2, 2개월 리밸·매월 9일, 비중캡 25% + 채권 폴백 + Ichimoku/ADX + 금 헤지, 거래비용 0.25%)**

| 지표 | 값 (2015 ~ 현재 백테스트) |
|---|---|
| CAGR (연복리) | 약 **27.4%** |
| 샤프 지수 | **1.42** |
| 최대 낙폭 (MDD, 일별 실측) | 약 **−20.1%** |
| 누적 총수익 | 약 1,450% |
| 연환산 알파 (vs KOSPI200) | 약 +10.5%/년 |

> 수치는 DB에 적재된 마지막 거래일까지로 자동 재계산되며 데이터 갱신 시 달라질 수 있습니다. 대시보드 "성과" 탭에서 현재 값을 확인하세요.

**측정 방식에 대한 중요한 주의 — MDD는 "일별 실측" 기준**
초기에는 리밸런싱 시점(반기)만 찍은 자산곡선으로 MDD를 계산해 구간 사이의 폭락(예: 2020 코로나)을 놓쳤습니다. 현재는 **매 거래일 자산곡선**으로 MDD(`mdd_daily`)를 계산합니다. 모든 MDD 논의는 이 일별 실측 기준입니다.

**손절을 쓰지 않는 이유** — 개별 종목 손절(−15%/−20%)을 백테스트하면 상관된 폭락에서 종목을 바닥에 잘라 손실만 확정하고 반등을 놓쳐 알파가 마이너스로 전환됐습니다. 그래서 손절 대신 **사전 분산(비중 캡)과 거시 방어(분할 스위칭)**로만 위험을 관리합니다.

백테스트 실행 예:
```bash
python main.py backtest --strategy ensemble --ai-weight 0.2 \
    --start-year 2015 --end-year 2024 \
    --market-split --trend-filter --market-cap-min 1e12 \
    --period-months 2
```

## 12. 시스템 검증 — 블랙스완·몬테카를로

`src/backtest/stress_test.py` — 실측 운용곡선(비중캡·채권·금 헤지 적용)을 입력으로 두 가지 검증을 합니다. 대시보드 "검증" 탭에서 확인합니다.

**① 블랙스완 방어 성적표** — 역사적 위기 3건에서 시스템 vs KOSPI200 비교:
- 2018 미중 무역전쟁/금리쇼크
- 2020 COVID-19 팬데믹
- 2022 인플레이션 대폭락

각 구간의 시스템 MDD·수익률·알파와 **방어율** = (1 − 시스템MDD/KOSPI MDD)×100 을 표로 보여줍니다.

**② 몬테카를로 10,000회 부트스트랩** — 일별 수익률을 5일 블록으로 무작위 재추출해 1만 개의 가상 타임라인을 만들고:
- 파산확률(원금 손실), 반토막 확률(−50%)
- CAGR 중앙값 / 1% 최악, MDD 1% 최악
- 참고용 1.5x·2.0x 스트레스(음수 수익률 증폭) 시나리오

**판정 체크리스트**(`verdict`): 실측 샤프 > 1.0, 일별 MDD > −30%, 표준 파산확률 < 5%, 1% 최악 CAGR > 0% 등 6개 항목 중 5개 이상 통과 시 합격.

## 15. 한계와 주의사항

1. **고성장 폭등장에서 시장 추종이 약함**: ROE 20%+ 원칙 때문에 ROE가 낮은 일부 대형주(예: 저ROE 구간의 삼성전자)를 제외합니다. 시장이 그런 종목 주도로 급등하면 벤치마크를 못 따라갈 수 있습니다. 이는 "원칙을 지키는 안정형 시스템"이라는 설계 선택입니다.
2. **MDD는 일별 실측 −20%대**: 분할 스위칭·금 헤지로 완화했지만 long-only 시스템의 구조적 한계가 있습니다.
3. **신규 분기 데이터 지연**: 한국 분기보고서 제출 기한이 분기말+45일이라, 가장 최근 분기는 늦게 채워져 후보 풀이 일시적으로 좁아질 수 있습니다.
4. **생존 편향**: pykrx는 상장 폐지 종목의 과거 데이터를 제한적으로만 제공해 일부 생존 편향이 남습니다.
5. **백테스트 ≠ 미래 수익 보장**: 모든 수치는 과거 데이터 기반 시뮬레이션입니다. 이 프로젝트는 **연구·교육용**이며 투자 권유가 아닙니다.
